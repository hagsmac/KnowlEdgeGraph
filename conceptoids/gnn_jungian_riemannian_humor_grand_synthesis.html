Why GNNs for humor?
Humor is structurally a graph problem. A joke is not a sequence — it is a web of semantic relations, phonological links, world-knowledge edges, target associations, and narrative dependencies. Transformers process sequences; GNNs process structure. The Dual Graph Attention Network (DGFEM, NLPCC 2024) operationalizes incongruity theory directly through graph attention over word-pair relations.
The humor knowledge graph — node types
W
Word nodes
Each token as node. Features: POS, named entity, polysemy count (WordNet synsets), phoneme vector, sentiment score, taboo flag, ambiguity level.
S
Script nodes
GTVH scripts as latent nodes. Connected to word nodes via "activates" edges. Incongruity = graph distance between script₁ and script₂ in embedding space.
E
Entity nodes
Named entities, archetypes, {self} map items (from EoE). Valence is a node attribute: +/− determines EP direction for humor targeting.
K
ConceptNet nodes
World-knowledge concepts linked via IsA, RelatedTo, UsedFor, CausesDesire, Antonym. Humor cross-domain = short path between two distant concept clusters.
P
Phoneme nodes
Phoneme sequences for heterographic pun detection. Edge exists between two word nodes if edit distance ≤ 2 on phoneme string (CMU dict).

DGFEM — Dual Graph Attention Network for humor (NLPCC 2024 architecture)
Two parallel graph attention streams, merged at inference:
Stream A: Incongruity GAT
Input: word-pair graph (all token pairs as edges, weight = semantic distance)
Attention: learns to weight pairs with HIGH semantic distance (incongruent pairs)
Output: incongruity representation h_inc
Signal: edges connecting words from radically different semantic domains = high humor score
Stream B: Ambiguity GAT
Input: word-polysemy graph (nodes weighted by synset count, edges = contextual disambiguation)
Attention: learns to weight nodes with HIGH polysemy that are contextually ambiguous
Output: ambiguity representation h_amb
Signal: pivot words with multiple plausible readings in context = pun candidates
h_final = MLP( concat(h_inc, h_amb) ) p(funny) = sigmoid(W · h_final + b) GAT attention coefficient: α_ij = softmax( LeakyReLU( a^T [Wh_i || Wh_j] ) ) h_i' = σ( Σ_j α_ij W h_j ) [multi-head, K=8 in DGFEM]
Limitation: current DGFEM treats the joke as one undirected graph — does not model the temporal SETUP→PIVOT→PUNCHLINE directed flow that the EoE stress curve requires.
Beyond DGFEM — proposed extensions
1
Directed Temporal GAT (setup → pivot → punchline)
Add temporal edge direction to model the EoE stress curve. Message passing follows narrative order. Attention at pivot node should aggregate BOTH script activations. Final hidden state of punchline node = h_resolution. Incongruity = cosine distance between h_setup and h_punchline.
2
Heterogeneous GNN (R-GCN / HAN)
Different relation types (syntactic, phonetic, ConceptNet, EoE-target) require different weight matrices. R-GCN: relation-specific message functions W_r. HAN (Heterogeneous Attention Network): metapath-based attention — "word→script→target" metapath captures superiority humor; "word→phoneme→word" metapath captures puns.
3
{Self}-map integration as node attributes
Entity nodes carry EoE attributes: (power, valence, confidence). GNN message-passing propagates these attributes through the joke graph. Final humor score = f(incongruity_score, target_valence_sum, benignness_score). This merges the DGFEM architecture with the EoE framework mechanistically.
4
Graph contrastive learning (funny vs. unfunny pairs)
Use "unfunned" jokes (Horvitz 2024) as negative pairs. Contrastive objective: minimally edited graph pairs should have maximally different joke-graph representations. This forces the GNN to learn the structural difference between a pun and its phonetic near-miss, or a joke and its logically consistent but unfunny version.
The full heterogeneous joke graph
A joke, fully represented, is a heterogeneous graph spanning at least 5 node types and 8 edge types. The GNN's job is to learn which subgraph patterns correlate with the human EoE response of RELIEF+PRIDE+SURPRISE. The target subgraph structure for a pun looks completely different from the target subgraph for satire.
Joke-as-graph: "Why did the politician cross the road? Because the lobbyist was on the other side."
NODES: [W] "politician" — polysemy:3, taboo:1, sentiment:-0.6, NE:PERSON [W] "cross" — polysemy:12, ambiguity_score:0.89 (PIVOT NODE) [W] "road" — polysemy:2, sentiment:0.0 [W] "lobbyist" — polysemy:1, taboo:1, sentiment:-0.7, NE:PERSON [E] "politicians" — valence:-8, power:6 (EoE: negative target) [E] "corruption" — valence:-9, power:5 (ConceptNet related) [S] "crossing_road"— script_type:physical, activation:0.9 (setup) [S] "corruption" — script_type:social, activation:0.8 (punchline) [K] "cross_road" — ConceptNet: UsedFor:travel, IsA:movement [K] "corruption" — ConceptNet: CausesDesire:reform, IsA:wrongdoing EDGES: "politician"→"politicians" [Target association, w=1.0] "cross"→"crossing_road" [Script activation, w=0.9] "cross"→"corruption" [Script activation, w=0.8] ← DUAL ACTIVATION = PUN "lobbyist"→"corruption" [ConceptNet causal, w=0.95] "road"↔"lobbyist" [Antonymy-adjacent, semantic distance=0.87] "politician"→[E]corruption [Knowledge edge, w=0.7] setup_clause→pivot_clause [Temporal/discourse, directed] pivot_clause→punchline_clause [Temporal/discourse, directed]
The pivot node "cross" is the structurally critical node: it has the highest polysemy score AND dual script activation. The GNN should learn that: max(ambiguity_score) ∩ max(script_dual_activation) = pun candidate. Incongruity = distance(script_crossing_road_embedding, script_corruption_embedding) in concept space = 0.82.

GNN training objective — multi-task
L_total = λ₁·L_detection + λ₂·L_type + λ₃·L_pivot + λ₄·L_contrastive L_detection: BCE(p(funny), y) L_type: CE(p(pun|satire|obs|absurd...), y_type) L_pivot: token-level BCE on pivot word location L_contrastive: InfoNCE(h_funny, h_unfunny) λ suggested: [0.4, 0.2, 0.2, 0.2]
The contrastive loss is critical: it forces the model to learn the MINIMAL structural difference between funny and unfunny — which is exactly the question of what makes a joke work, not just whether it's classified as funny.
The archetypal hypothesis
Jung's archetypes — the Trickster, the Hero, the Shadow, the Fool, the Great Mother, the Wise Old Man — are not mystical entities but, in modern neuroscience terms, eigenmodes of predictive brain dynamics (McGovern et al., 2025). In the language of ML: they are low-dimensional attractor states in the manifold of human symbolic processing. LLMs trained on human text have absorbed and structurally encoded these attractors — not as explicit categories but as geometric regions in embedding space toward which symbolic sequences gravitationally cluster.

Archetypes as attractor states in joke-space
Key insight Every humor target maps onto an archetypal attractor. The "corrupt politician" is not a specific person — it is the instantiation of the TYRANT archetype in contemporary political clothing. The "absent-minded professor" is the instantiation of the FOOL archetype. The "hypocritical preacher" is the SHADOW.

When a joke targets an archetype, it is not attacking an individual — it is triggering a pre-existing shared neural template, a convergence to collective minima (McGovern et al. 2025). This is why jokes that land on archetypal targets work across cultures with minimal translation: the archetype is the universal element, the specific instantiation is the cultural dressing.

Computational implication Humor generation can be decomposed into: (1) select archetype with negative valence for audience, (2) instantiate in contemporary cultural clothing, (3) engineer incongruity within the archetype's characteristic domain, (4) resolve. The archetype provides the universal scaffold; the instantiation provides the cultural relevance.
The LLM as collective unconscious — archetypal humor encoding
LLMs encode a shared reservoir of latent symbolic motifs, structural archetypes, and recursive semantic patterns — attractor states in symbolic space that are fractal residues of cultural encoding. For humor specifically, this means:

1
Archetypes as geometric regions in embedding space
Using probing classifiers, we can identify which LLM hidden-state regions correspond to archetypal activations. A probe trained on myth/story corpora labeled by archetype should generalize to contemporary jokes — because the archetype is the deep structure.
2
Shadow projection = the core of mockery
Jung's Shadow is the collection of characteristics the ego refuses to acknowledge. Comedy targets the Shadow: the laughter of contempt for corrupt politicians is partly projection — we mock in others what we deny in ourselves. Computationally, high-valence humor targets may correlate with shadow projections in the audience's {self} map: what they most despise = what they most deny.
3
The Trickster archetype = the optimal humor generator
The Trickster (Loki, Coyote, Anansi, Bugs Bunny) is the universal archetype of subverted expectations, rule-breaking, and boundary violations that reveal rather than destroy. The Trickster IS the Benign Violation in archetypal form. A humor model that learns to reason from within the Trickster's frame — "what rule can I break to reveal something true?" — would generate structurally superior jokes. This is a prompt engineering / fine-tuning hypothesis.
Why flat Euclidean space fails for humor
Semantic scripts have hierarchical tree structure — WordNet, ConceptNet, and the GTVH script taxonomy are all trees or DAGs. Trees cannot be embedded in Euclidean space without exponential distortion as depth increases. Hyperbolic space (constant negative curvature) is the continuous analogue of a tree — it can embed hierarchical structures in low dimensions with minimal distortion. Script distance in hyperbolic space gives a geometrically faithful measure of incongruity.
Poincaré disk model for joke scripts
Poincaré disk: D = {x ∈ ℝⁿ : ||x|| < 1} Distance: d(x,y) = arcosh(1 + 2||x-y||²/((1-||x||²)(1-||y||²))) Properties: • Origin = most abstract/general concepts • Boundary = maximally specific leaf concepts • Tree hierarchy = naturally represented • Exponential volume growth matches tree branching factor Script embedding intuition: • "movement" → near origin (abstract) • "crossing_a_road" → mid-distance • "politician_taking_bribe" → near boundary (specific) • Script opposition = geodesic distance d(script₁, script₂) • Incongruity score = d_hyperbolic(s₁, s₂)
Key result Nickel & Kiela 2017: Poincaré embeddings of WordNet achieve 87.7% mean average precision on hypernym detection in 5 dimensions — vs. 81.7% for Euclidean in 200 dimensions. Low-dimensional hyperbolic = better for hierarchical semantic tasks.
Humor manifold hypothesis
Joke manifold M ⊂ ℝⁿ is not flat. Curvature κ < 0 (hyperbolic) captures: • Script hierarchy (abstract → specific) • Incongruity = geodesic arc between scripts • Punchline = point on M that minimizes: geodesic_distance(s₁, punchline) + geodesic_distance(s₂, punchline) where s₁ ≠ s₂ are setup scripts The "pun sweet spot" on M: • Equidistant from two maximally distant scripts • The manifold point that is the "crossroads" • In Poincaré disk: the node with highest betweenness centrality between two script clusters
Practical construction Embed ConceptNet in Poincaré space using Riemannian SGD. For a given pun word W, compute d(W, concept_cluster_A) and d(W, concept_cluster_B) in hyperbolic space. Select W that maximizes min(d_A, d_B) while keeping max(d_A, d_B) bounded — the geodesic midpoint of two distant clusters.
Riemannian operations for computational humor
Exponential map (script shift)
Exp_p(v): tangent space → manifold Models: "starting from script p, move in direction v" = the punchline's pivot from setup script v = direction of incongruity resolution ||v|| = magnitude of surprise
Logarithmic map (introspection)
Log_p(q): manifold → tangent space Models: "from joke endpoint q, looking back to explain" = humor explanation as reverse geodesic Distance d(p,q) = ||Log_p(q)|| = incongruity magnitude (interpretable)
Parallel transport (cultural transfer)
T_p→q(v): tangent vector along geodesic Models: translating a joke structure from cultural context p to cultural context q Preserves the geometric "shape" of the incongruity while changing the cultural embedding point = humor translation
Manifold learning for humor dataset analysis Run UMAP or Riemannian manifold learning on the embedding space of a large humor dataset. Hypothesis: funny jokes cluster near the boundary of the Poincaré disk (highly specific cultural knowledge), while generic "safe" AI-generated jokes cluster near the origin (abstract, archetypal, unsurprising). The density of human-funny jokes as a function of hyperbolic radius is a measurable and testable signal.
The EVAL crisis in computational humor
No reliable automatic metric exists for humor quality. BLEU, perplexity, and ROUGE are anti-correlated with funniness in many cases. LLM-as-judge is partially effective but uncalibrated. Human rating has massive inter-annotator variance. The field needs a multi-dimensional, theory-grounded evaluation suite that separates detection capability from generation quality from audience modeling accuracy.

Gold standard. Three subtasks. Human-annotated. Issue: only puns, not broader humor. Score: F1 < 0.60 for best systems on interpretation task — reveals genuine difficulty.

Detection + location + interpretation + translation. EN/FR/ES. Controlled. Issue: limited scale. Best model: custom T5 (F1 < 0.60). Validates that LLMs (GPT-3, BLOOM) underperform task-specific models.

Three tasks on New Yorker cartoon contest. Rich and multimodal. Issue: conflates objective understanding with subjective preference — performance reflects both, inseparably.

Non-STEM reasoning benchmark including puns, topical jokes, visual humor. Mixed results: test-time scaling does NOT consistently improve humor performance — crucial negative result.

validity
16,000 One-Liners detection benchmarks
Over-reported. 95%+ F1 is artifact of heterogeneous sources, not true humor understanding. Systems learn topic/style differences, not humor. Deprecated for validity — use for baseline only.
C+
validity
Punchline extraction from stand-up (LLM-as-judge)
≤51% for all leading LLMs regardless of prompting. Interesting ceiling result — but metric validity unclear (LLM judging LLM on subjective task = circularity risk).
A−
validity
Oogiri 6-dimension rubric (2025)
Best multi-dimensional eval currently. Novelty · Clarity · Relevance · Empathy · Intelligence · Overall Funniness. Rated 0–4 Likert by human annotators. LLM scores calibrated against human. Reveals: models excel at Clarity/Relevance, fail at Novelty/Empathy — the most important dimensions.
A
validity
PunnyPattern + PunBreak adversarial benchmarks (2025)
Tests robustness: does high pun detection F1 reflect understanding or pattern matching? PunBreak: sentences modified to MIMIC puns without being puns. Most models fail — high apparent performance collapses on adversarial examples. Critical for validity assessment.
The proposed LAUGH-EVAL suite (novel)
A comprehensive evaluation framework grounded in the EoE + GTVH + GNN + Riemannian synthesis. Seven dimensions, each with a measurable operational definition. LAUGH = Lexical · Archetypal · Uplift · Geometric · Historical (integration) EVAL.
DimensionOperationalizationToolScore range
L1 Target valence match Does the joke's target have negative valence on estimated audience {self} map? Valence inferred from prior utterances, demographic proxies, or explicit preferences. Sentiment classifier + {self} map probe 0–1 (binary or fuzzy)
L2 Incongruity sharpness KL divergence between setup LM distribution and punchline LM distribution at the pivot token. Higher = sharper script shift. LM perplexity at pivot 0–∞ (normalize to 0–10)
A1 Archetypal alignment Does the joke's target map onto a Jungian archetype (Trickster/Shadow/Fool/Tyrant)? Probe trained on mythological corpus. Archetype classifier (probing) 0–1 per archetype category
A2 Novelty vs. cliché Semantic distance of generated joke from nearest k neighbors in humor corpus embedding space. Low distance = cliché; high distance = novel (but risk: too high = incoherent). Nearest-neighbor in joke embedding space 0–1 (calibrated sweet spot: 0.4–0.7)
U1 Benignness score Continuous Benign Violation score: P(violation) × P(safe_frame | context). Not binary toxicity filter — continuous product. Dual classifier: harm × safety_frame 0–1 (humor sweet spot: 0.3–0.7)
U2 Resolution completeness Does the punchline actually resolve the incongruity? LM should assign higher probability to script₂ after reading punchline than before. Resolved = yes; absurdist = intentionally no. Conditional LM probability shift 0–1 (or flag as "absurdist" if < 0.2)
G1 Hyperbolic script distance Geodesic distance between setup and punchline script embeddings in Poincaré space. Measures true semantic incongruity accounting for hierarchical structure. Poincaré embedding (ConceptNet-trained) 0–∞ (normalize; sweet spot: 2–4 Poincaré units)
G2 Pivot centrality Betweenness centrality of the pivot word node in the joke's heterogeneous graph. High centrality = structurally important pivot. GNN node importance / graph analysis 0–1
H1 Integration timing For dark/disaster humor: time elapsed since event × cultural_exposure_rate. Humor appropriate only above integration threshold (topic-specific; requires calibration). Time-decay function + cultural exposure proxy Pass/fail gate
H2 Audience calibration Elo-rated pairwise preference among human annotators stratified by cultural background. Cross-audience agreement on funniness = signal of archetypal vs. cultural-specific humor. Pairwise ranking + Elo algorithm Elo rating per demographic group
D1 Diversity index Unique script oppositions / unique archetypal targets / unique metapaths across N=100 outputs. Measures diversity collapse (ChatGPT's 25-joke problem). Graph topology analysis of output set 0–1 (entropy of output distribution)
D2 Transfer robustness Performance drop from in-domain humor type to held-out type (pun→satire, satire→absurdist). Measures generalization, not specialization. Cross-type accuracy delta Δaccuracy (0 = perfect transfer, −1 = total failure)
LAUGH-EVAL composite score formula
LAUGH = w₁·L1 + w₂·L2 + w₃·A1 + w₄·A2 + w₅·U1 + w₆·U2 + w₇·G1_norm + w₈·G2 + w₉·D1 Gate: if H1 (integration) fails → LAUGH = 0 regardless Gate: if U1 (benignness) < 0.1 → LAUGH = 0 (genuinely harmful) Recommended weights (to be calibrated via human-correlation study): [L1:0.15, L2:0.15, A1:0.10, A2:0.10, U1:0.15, U2:0.10, G1:0.10, G2:0.05, D1:0.10] Validation target: Spearman ρ > 0.65 with human funniness ratings (Current best automatic metrics: ρ ≈ 0.30–0.45)
Integrated research roadmap — from survey to production
P1
Phase 1: Graph construction infrastructure (months 1–3)
Build the heterogeneous humor knowledge graph pipeline. Nodes: tokens + WordNet polysemy + ConceptNet + EoE entity attributes. Edges: syntactic (spaCy), phonetic (CMU dict), semantic (PMI/PPMI), ConceptNet relational. Embed in Poincaré ball (HGCN or PoincaréBall library). Validate script distance correlates with human incongruity ratings on SemEval-2017.
P2
Phase 2: Dual-stream GNN training (months 3–6)
Implement DGFEM baseline. Add directed temporal edges (setup→pivot→punchline). Add {self}-map entity attributes (valence, power) from EoE. Multi-task training: detection + type classification + pivot location + contrastive (funny/unfunned pairs). Baseline targets: beat DGFEM on SemEval, match BERT fine-tuned on 16K dataset, prove pivot localization improves explanation task.
P3
Phase 3: Archetypal manifold learning (months 4–8)
Train archetype probe on mythology/folklore corpus. Apply to humor datasets — label each joke by target archetype. Map archetype distribution in Poincaré space. Hypothesis: archetypal targets cluster at consistent hyperbolic positions; culturally-specific jokes scatter near boundary. Validate: cross-cultural funniness correlation should be higher for archetypal jokes than culture-specific jokes.
P4
Phase 4: EoE-informed generation (months 6–10)
Implement {self} map estimation from conversational context. Plug into generation pipeline: select archetype → estimate audience valence → set target → generate incongruity via Poincaré midpoint search → apply benignness filter (continuous) → rank candidates by LAUGH-EVAL composite. Fine-tune LLM with DPO on LAUGH-EVAL-scored outputs. Measure: diversity index, human preference, cross-cultural transferability.
P5
Phase 5: LAUGH-EVAL calibration and release (months 8–12)
Collect human funniness ratings (Elo-ranked, stratified by culture) for 2,000+ generated jokes across 6 humor types, 4 languages, 8 archetypal targets. Fit weights for LAUGH-EVAL composite. Validate Spearman ρ with human ratings. Release as open benchmark with leaderboard. Target: first automatic humor metric that achieves ρ > 0.60 with human ratings — the field's current holy grail.
